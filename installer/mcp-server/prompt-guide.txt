INFLUXDB QUERY FILE LIMIT AND MEMORY LIMITS (CRITICAL):
- InfluxDB has a hard limit of 432 parquet files per query.
- Multi-day queries WILL fail with "Query would exceed file limit" error.
- Wide-schema queries with many columns (e.g. TORCH battery: 5 modules × many cell signals)
  can hit memory exhaustion ("Not enough memory to continue external sort") with windows > 2h.
- ALWAYS use 1-hour windows for multi-day scans, especially for battery/TORCH queries.
- Wrap each window in try/except — log errors and continue rather than crashing:
    from datetime import datetime, timedelta
    import pandas as pd

    start, end, window_hours = datetime(2025, 10, 4, 12), datetime(2025, 10, 5, 0), 1
    frames, cur = [], start
    while cur < end:
        nxt = min(cur + timedelta(hours=window_hours), end)
        try:
            chunk = slicks.fetch_telemetry(cur, nxt, signals=["INV_Motor_Speed"],
                                           schema="wide", filter_movement=False)
            if chunk is not None and not chunk.empty:
                frames.append(chunk)
        except Exception as e:
            print(f"Error fetching {cur} - {nxt}: {e}")
        cur = nxt
    df = pd.concat(frames) if frames else pd.DataFrame()

--- slicks.connect_influxdb3 ---
slicks.connect_influxdb3(db=None, table=None, url=None, token=None, org=None, schema=None)
  Set the active database before any fetch call. db and table are usually the same value.
  Env vars (INFLUX_URL, INFLUX_TOKEN, INFLUX_DB) are already set — only pass db/table.
  Example:
      slicks.connect_influxdb3(db="WFR25", table="WFR25")

--- slicks.fetch_telemetry ---
slicks.fetch_telemetry(start_time, end_time, signals=None, client=None,
                       filter_movement=True, resample="1s", schema="wide")
  Fetch one window of data. Returns a pandas DataFrame (DatetimeIndex, UTC tz-naive)
  or None if no data. Columns are signal names.
  - signals: list of column names, or None for default set
  - filter_movement: True filters to rows where the car is moving
  - resample: pandas freq string e.g. "1s", "100ms", None for raw
  - schema: always "wide" — never use "narrow"
  WARNING: use windowed loops (1h windows) for multi-day ranges; single large queries OOM.

--- slicks.fetch_telemetry_chunked ---
slicks.fetch_telemetry_chunked(start_time, end_time, signals=None, client=None,
                                filter_movement=True, resample="1s",
                                chunk_size=timedelta(hours=6), max_workers=1,
                                show_progress=True, schema="wide")
  Same as fetch_telemetry but automatically splits on parquet file limit errors.
  Prefer this for ranges > 2 hours when you don't want to write your own loop.
  Returns combined DataFrame or None.

--- slicks.discover_sensors ---
slicks.discover_sensors(start_time, end_time, schema="wide")
  Returns sorted list of all column names in the database. For wide schema this is
  an instant metadata query (ignores time range). Use to find exact signal names.
  Example:
      sensors = slicks.discover_sensors(datetime(2025,10,4,12), datetime(2025,10,4,18))
      torch_cells = [s for s in sensors if "Cell" in s and "Voltage" in s]

--- slicks.scan_data_availability ---
slicks.scan_data_availability(start, end, timezone="America/Toronto",
                               table=None, bin_size="1h", include_counts=False)
  Find all time windows that have data in a date range. Returns a ScanResult object.
  result.calendar_view() → matplotlib Figure (save with fig.savefig("calendar.png"))
  Example:
      result = slicks.scan_data_availability(datetime(2025,8,1), datetime(2025,11,1))
      fig = result.calendar_view()
      fig.savefig("calendar.png")
